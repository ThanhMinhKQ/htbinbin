# UI/UX Rules

- Preserve Vietnamese labels, hotel-domain terms, and existing tone unless task asks for copy changes.
- PMS/inventory screens must stay usable on common laptop widths and mobile-width checks when layout changes.
- Keep loading, empty, error, and success states visible for user-triggered actions.
- Do not hide destructive or irreversible actions behind ambiguous buttons.
- For Jinja template changes, verify the rendered page path when credentials/test data are available.
- For CSS changes, check dark/light theme interaction when affected files include PMS or reservation hub styles.
